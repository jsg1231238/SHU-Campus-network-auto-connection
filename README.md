# 校园网自动认证脚本（Ruijie ePortal）

用一条 HTTP 请求代替"开浏览器、模拟点击"的校园网认证方式，可挂 Windows 计划任务无人值守运行——开机自动登录，掉线自动重连，全程无窗口、锁屏也能跑。

已在**上海大学（SHU）校园网**（锐捷 / Ruijie ePortal）验证可用。其它学校若用的也是锐捷 ePortal，大概率只需改几个参数即可复用；具体见下方「不同学校怎么办」。

---

## 这是什么 / 为什么做这个

校园网每天会强制掉线一次，手动重连很烦。最初的方案是写脚本自动打开浏览器、模拟点击登录按钮，但这类 UI 自动化非常脆弱（依赖窗口焦点、分辨率、界面文字，锁屏或没人在电脑前就完全失效）。

这个项目把认证过程逆向清楚后，改成**直接向认证服务器发送一次登录请求**——不开浏览器、没有任何窗口，锁屏、计划任务后台运行都没问题。核心难点是复刻了认证系统一套非标准的 RSA 密码加密逻辑，细节见下方原理文档。

## 快速开始

**给别人用 / 给自己配置，看这份**：[给同学使用-操作指南.md](给同学使用-操作指南.md)

里面分两种情况：
- **同校同网络**（比如同学之间）：只需改 WiFi 名和账号两处，5 步搞定。
- **不同学校 / 不同认证系统**：不能直接套用，需要重新走一遍逆向分析流程（方法见下方原理文档）。

如果你就是要直接把整份工具转发给别人、让对方少改一步，也可以直接发 [分享给同学](分享给同学) 这个文件夹（脚本 + 启动器 + 精简说明，已用占位符替换个人信息）。

## 文件说明

| 文件 / 目录 | 作用 |
|---|---|
| `campus-net-api.ps1` | 主脚本，核心认证逻辑 |
| `local.settings.ps1`（不在仓库里） | 你自己的 WiFi 名 / 账号，本地私有，不进版本库，运行时自动加载覆盖 |
| `launch-hidden.vbs` | 静默启动器，配合计划任务实现无窗口后台运行 |
| `上网认证_files/` | 逆向分析用的认证系统前端 JS（已确认不含个人信息，作为逻辑可验证的"证据"保留） |
| `分享给同学/` | 精简打包版，方便直接转发给同校同学 |
| `diag-portal.ps1` | 排查网关重定向问题用的诊断脚本 |

## 安全说明

- 密码通过 Windows DPAPI 加密后存本地文件，脚本本体不含任何明文密码或个人信息。
- `.gitignore` 已排除本地私有配置、运行日志、加密密码文件等，仓库里的一切内容均已核对确认不含真实个人数据。
- 仅用于自动化本人已有合法权限的校园网认证登录，不涉及绕过认证或未授权访问。

## 免责声明

本项目基于对认证页面前端代码的公开分析实现，仅供个人自动化登录使用。不同学校 / 不同时期的认证系统实现可能不同，使用前请确认自己有合法的账号访问权限，并遵守所在学校的网络使用规定。
