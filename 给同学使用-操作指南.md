# 校园网自动认证脚本 · 给别人用的操作指南

这份脚本能让电脑**自动**帮你完成校园网登录，不用每次掉线了手动打开网页输密码。前提：你和"原作者"用的是**同一个学校、同一套认证系统**。

先判断你属于哪种情况——两种情况的处理方法完全不同，别搞混。

---

## 第一步：判断你能不能直接用这份脚本

打开浏览器，**断开校园网认证**（或者等它自动掉线），随便打开一个网站，会被强制跳转到一个"登录校园网"的页面。看这个页面的**网址栏**：

- 如果网址是 `http://10.10.9.9/eportal/...` 开头，页面样子和下面这张描述一致（深蓝色背景、"欢迎登录校园网"字样、账号密码两个输入框、右上角有"校园网"和"统一身份认证"两个切换）→ **大概率是同一所学校、同一套系统，走"情况 A"**。
- 如果网址是别的 IP、页面样式完全不同 → **不是同一套系统，走"情况 B"**。

拿不准的话，把认证页面截图发给对方（写这份脚本的人）确认一下最快。

---

## 情况 A：同校同网络——只需改 3 处

脚本里绝大部分内容（登录服务器地址、加密公钥、请求格式）是**这个学校认证系统本身决定的**，跟你是谁没关系，所以不用动。真正跟"你个人"有关的只有 3 样东西：

| 需要改的东西 | 是什么 |
|---|---|
| 你的 WiFi 名称 | 你连接的无线网络名（SSID） |
| 你的校园网账号 | 一般是学号 |
| 你的校园网密码 | 会用加密方式单独存，不会明文写进脚本 |

### 具体步骤

**① 拿到整个文件夹**
对方会把整个项目文件夹发给你（或者你去他电脑上拷一份），里面至少要有这三个文件：
- `campus-net-api.ps1` —— 主脚本，真正干活的程序
- `launch-hidden.vbs` —— 让脚本在后台安静运行、不弹黑窗口用的小工具
- （`.campuspwd` 如果对方不小心也给了你，**直接删掉**——那是他的密码，换到你电脑上根本打不开，留着也没用，删了干净）

把这个文件夹放到你电脑上任意位置，比如 `D:\campus-net\`。

**② 用记事本或 Typora 打开 `campus-net-api.ps1`，改开头几行**

找到最上面这几行（大概第 11–16 行）：

```powershell
param(
    [string]$WifiSSID   = "改成你的WiFi名字",     # ← 改成你的 WiFi 名字
    [string]$Account    = "改成你的校园网账号(学号)",  # ← 改成你的学号
    [string]$Password   = "",
    [string]$Service    = "shu",              # ← 这个不用改，是学校统一的
    [string]$PortalHost = "10.10.9.9",        # ← 这个也不用改
```

只改 `$WifiSSID` 和 `$Account` 后面双引号里的内容，其余不动。**`$PubExp`、`$PubMod`（一长串字母数字）也完全不用碰**——那是认证服务器的公开"锁头"，全校所有人共用同一个，不因人而异。

**③ 存一次你自己的密码（只需做一次）**

打开 **PowerShell**（开始菜单搜"PowerShell"，直接回车打开就行，不用管理员权限）。

依次做：
1. 输入 `cd ` 后面**空一格**，再把你放脚本的文件夹路径拖进这个黑窗口（会自动填路径），回车。效果类似：
   ```
   cd D:\campus-net
   ```
2. 复制粘贴下面这行，回车：
   ```powershell
   (Read-Host '校园网密码' -AsSecureString | ConvertFrom-SecureString) | Set-Content .\.campuspwd -Encoding ASCII
   ```
3. 它会提示你"校园网密码:"，输入你的密码（**输入时屏幕上不会显示任何字符，这是正常的，不是没反应**），输完直接回车。
4. 这一步做完后，文件夹里会多出一个叫 `.campuspwd` 的文件——这是你的密码经过 Windows 自带的加密功能（DPAPI）处理后的结果，**只有你这台电脑、你这个 Windows 账户能打开它**，就算文件被别人拷走也没用，所以可以放心留在文件夹里。

**④ 测试一遍（确认真的能登录）**

先确保你现在其实是"未登录"校园网的状态（可以先在浏览器里手动登录一次校园网，再点页面上的"注销"按钮）。然后在刚才那个 PowerShell 窗口里输入：

```powershell
powershell -ExecutionPolicy Bypass -File .\campus-net-api.ps1
Get-Content .\campus-net-log.txt -Tail 3
```

第二条命令会显示最后 3 行日志。如果看到 `SUCCESS`，说明成功了，现在应该已经能上网了。

如果看到 `密码错误` 或别的报错，**不要自己瞎改**，把这三行日志截图发给写这份脚本的人，让他帮你看（大概率是账号输错，或者你这个学校的服务器和对方那份脚本假设的不完全一样）。

**⑤ 设置成"自动运行"（这样以后就不用管了）**

在开始菜单搜索 **PowerShell**，右键点它，选**"以管理员身份运行"**（这一步必须用管理员权限，否则设置不上）。在弹出的蓝色/黑色窗口里，把下面这一整段复制粘贴进去，回车执行（记得把路径改成你自己文件夹的实际路径）：

```powershell
$vbs = "D:\campus-net\launch-hidden.vbs"
$act = New-ScheduledTaskAction -Execute "wscript.exe" -Argument "//B //Nologo `"$vbs`""
$t1  = New-ScheduledTaskTrigger -AtLogOn
$t2  = New-ScheduledTaskTrigger -Once -At (Get-Date) -RepetitionInterval (New-TimeSpan -Minutes 30) -RepetitionDuration (New-TimeSpan -Days 3650)
$set = New-ScheduledTaskSettingsSet -AllowStartIfOnBatteries -DontStopIfGoingOnBatteries -StartWhenAvailable -ExecutionTimeLimit (New-TimeSpan -Minutes 3) -MultipleInstances IgnoreNew
Register-ScheduledTask -TaskName "CampusNetAutoLogin" -Action $act -Trigger $t1,$t2 -Settings $set -Description "校园网自动认证"
```

**还要改一个地方**：用记事本打开 `launch-hidden.vbs`，把里面 `E:\claude-work\20260709_netconnect\campus-net-api.ps1` 这一段路径，也改成你自己文件夹里 `campus-net-api.ps1` 的实际完整路径，保存。

做完这些，以后你的电脑会在**每次开机登录 + 之后每 30 分钟**自动检查一次网络，掉线了就自动重新登录，全程没有任何弹窗，你不用管它。

## 情况 B：不同学校 / 不同认证系统——不能直接套用

如果你连的不是这套系统（网址不是 `10.10.9.9`，页面长得也不一样），说明背后的认证服务器实现方式可能完全不同：

- 有的是另一个厂商（比如"深澜"、"H3C"）而不是这里用的"锐捷"；
- 就算同是锐捷，密码加密的"锁头"（公钥）、服务器地址、字段名也可能不同。

**这种情况不能靠改几个变量解决**，需要重新走一遍"侦查"过程：把你们学校认证页面的网页文件（HTML）和相关脚本文件（JS）保存下来，分析它到底怎么跟服务器"对话"、密码怎么加密的，再照着写一份新脚本。

这份项目里的 [校园网认证-原理讲解.md](校园网认证-原理讲解.md) 完整记录了这套方法（怎么用浏览器"开发者工具"抓包、怎么找加密逻辑、怎么一步步排错），可以拿给会写代码、或者愿意用 AI 助手（比如 Claude Code）重新分析的人参考，照着这个方法论对你们学校的系统重新做一遍即可。**不建议直接把这份 `campus-net-api.ps1` 改一改就用**，大概率跑不通，还会浪费时间排错。

---

## 安全提醒

- 打包文件夹发给别人时，**记得删掉 `.campuspwd` 和 `campus-net-log.txt`** 这两个文件（前者是密码，后者的日志里可能残留一些会话信息）。就算不小心带过去了，`.campuspwd` 在别人电脑上也解不开、用不了，但删干净总是更保险的习惯。
- 不要把这两个文件通过微信/QQ 等聊天工具直接发送或截图。
